命名规范

Rust目前的命名规范还没有完全设计好，但基本记忆方法就是抽象没有实体的东西使用大驼峰，不变的东西用全大写，其它的东西用全小写。当然具体怎么个命名规范，还是等用到的时候再看吧，等用多了一般也就习惯记住了。

除此之外，我们在设计程序的时候可能会设计出一个没用的变量，这个变量之所以没用是因为你还在写程序或者设计程序中，它可能在未来有用，但是现在只是放在这里做一个记录。那么你可以在这个变量的开头加一个下划线_，如果不加，Rust的编译器将会给你个警告。当然这个你没记住也没关系，vscode的rust插件会提醒你的。

变量遮蔽

在Rust中，变量遮蔽是指定义新变量时，如果变量名与已有变量名相同，则新变量将覆盖（遮蔽）原有变量。变量遮蔽的主要作用是在特定作用域中使用不同的变量名来表示不同的含义，而不会影响到其他作用域或全局作用域中的变量。变量遮蔽可以用来：

  1. 避免命名冲突
  2. 增强代码可读性（治疗命名困难症）
  3. 实现局部变量的修改而不对外部造成影响

总体给我的感觉有点类似于C语言局部变量和形参的结合体。
