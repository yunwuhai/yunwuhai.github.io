整数类型

现在进入2.2.1的内容。

Rust的数值类型采用了比较舒服的位数表示法，个人认为这种表示方法在针对一些内存操作的时候会更加精确，也更加不容易出问题。通过如i和u表示是否有符号，通过数字表示表示比特位数，比如u8就表示无符号八位整数。

值得注意的是，Rust目前语法直接支持的最长整数长度为128位。此外Rust的整数类型中还有一个叫isize和usize的类型，这种类型会根据程序运行的CPU位数来决定。比如我们常见的64位计算机，编译这个计算机上的程序时，这个类型会被编译为64位。

同时，在Rust中可以使用十进制、十六进制、八进制和二进制来表示数字，表示方法和C语言相同。

但是值得注意的是，Rust的数字中间可以添加数字分隔符，也就是用一个下划线来分隔一长串数字，使其更加易读，例如82012可以写成82_012。这个分隔符可以在上述四种进制中使用，且不怎么限制间隔数字，像是8_2_0_1_2从理论上来说也是允许的，但注意不能在数字开头和结尾出现，也不能重复使用。这个语法是C语言的C11标准之前所没有的，所以这里提一嘴。

此外，有时候我们需要使用ASCII码的码值，只要数据类型是u8，你就可以给它绑定类似于b'A'的ASCII值，这里表示的作用其实就是对A取它的二进制字面量。

Rust的整型默认使用i32。

此外Rust需要对数据溢出进行处理，在Debug模式下溢出会认为是错误的而崩溃，而在Release模式下溢出会默认按照补码循环溢出。当然也可以显式地表示溢出，这个需要用的时候再搜索就行。

浮点类型

Rust中浮点类型有两种基本类型，f32和f64，默认是f64，其现代计算机中其速度和f32几乎相同但精度更高，感觉相当于C语言中的float和double，其内存实现方法符合IEEE-754标准。

浮点数存在浮点数陷阱，想必各位在学其他语言的时候应该听说过那个传说中的0.1+0.2!=0.3的例子，在教程里也说了浮点数陷阱的相关内容，但是感觉讲得有点绕，我让ChatGPT重新说了一遍，感觉用这个理解和避免的方法会更好一点。

反正说白了，如果你希望进行特别精密的计算，就少用浮点数，如果能够在一定位数内解决问题，可以采用整数来计算会更好（反正就是乘除几个10的差别）。不过现在好像也有一些硬件支持取硬浮点数了，那种计算会更加精确和快速一点，比如说一些带有FPU的单片机，感觉有可能有些显卡在浮点数运算加速上也是这种类似的方法，但这终究是少数，总之等用到的时候请自行取了解和体会。

对于一些计算，可能产生数学未定义的内容，例如对负数取平方根或者除数为0这种情况，一般来说，这种计算在Rust中，其返回值为NaN（Not a Number)，NaN无法做任何比较或计算，出于防御性编程的考虑，可以使用is_nan()等方法来判断一个数值是不是NaN。

数字运算

加减乘除的写法和其它语言基本一样，不解释了。但是你需要注意的是Rust语言在计算的时候必须保证是同类型的数据在进行计算，也就是说u32只能和u32的数据进行计算，和C不同，它不存在隐式转换。

位运算

Rust也存在位运算，毕竟这个语言是奔着系统级编程的目标来的，而位运算在系统级编程或硬件编程中非常有用。不过这里也没有什么好说的，Rust的位运算和C语言的使用方法基本一样。

换书

写到这里，我已经感觉非常疲惫，我不否认《Rust语言圣经》是一篇非常不错的教程书，至少从它的语言风趣和内容详实以及更新非常勤可以看出。

但是根据我现在的学习，我感觉它更加适合作为一个已有一定Rust开发经验的从业人员的入门教程，但如果希望特别详细准确地零基础学习到Rust语言的朋友，这本书并不太适合，甚至可以说一些语法突然冒出来但不说明这种语法的使用方法仅仅说它有什么用，实在有点劝退。

为此，我决定改用Rust官方的教程书《The Rust Programming Language》和《Rust by Example》，等基础学好了再重新看Rust语言圣经。
