第一章-硬件基础

发表于 2022-12-31 更新于 2023-02-28 分类于 编程教程

开篇碎碎念

我写这样一个教程的主要目的其实只是为了检验自己的学习情况，同时做个简单的记录方便之后回顾。但同时也是希望用一种我认为更加合理的方法来讲述如何学习任何一门编程语言，能够方便到其他有需要的朋友。整个教程将按照以下顺序进行讲述：

  1. 硬件基础：这个部分我将以很简单的电路知识为引子，对程序在硬件层面所作工作做一个简单介绍，方便之后的理解。
  2. 指令和数据：有了硬件基础后，我们已经可以大致明白一个程序的工作流程，在这一部分我们将从指令与数据的调用层面，来理解一个编程语言的设计理念。
  3. 基础语法：在已经明白了指令和数据的调用后，我们的脑海中自然而然会构想出如何设计出一门语言，无论如何构想，但是有一些基础的设计总是相似的，你几乎可以在任何一门语言上看到和这些功能相关的语法设计。
  4. 语法糖：有了基础语法后，一门语言想要脱颖而出往往离不开对一些功能的简化写法，它们被称为语法糖。不同语言的语法糖总是千奇百怪的，不可能面面俱到，故将着重围绕C++来介绍它的语法糖，而其它语言的语法糖可能有所差别，但设计理念总是大差不差的。
  5. 语言的差异：不同的语言之间肯定是存在差异的，除了基础语法和语法糖的设计不同外，面向对象和面向过程、解释语言和编译语言等内容也常常是不同语言之间性能、难度、写法存在差异的根源。不过这个部分涉及面太过广泛，以我的知识水平不可能面面俱到，更多还需各位在实践中慢慢学习体会。

以上五个部分将以C++为主线，中间穿插其它语言的内容，它并不能让人直接成为编程高手，但是对于编程的入门而言我想应该会有很大帮助。

现在，我们进入第一章的正题。

硬件基础

开关与信号

这是一个简单的带开关LED电路：

VCC是电源输出，GND是电源接地，KEY是一个开关，LED是一个发光二极管。当开关闭合时，发光二极管会发光，断开时LED会熄灭（当然，事实上正常情况下这个电路还需要加上一个电阻来防止LED烧坏，但是我们现在只需要关注它的开关和亮灭，所以不要在意麻烦的细节）。

现在，通过改变开关的开合，我们就可以控制LED的亮灭，此时LED拥有两种状态，如果我们规定，每过一秒整的时候，可以按一下开关，那么通过连续监测LED的状态，我们就可以实现信号的传递了。

而为了方便我们在信号发送和接收时的表示，我们可以用两个数字来分别代表不同的状态，比如开关断开和LED熄灭的状态我们用0来表示，开关闭合和LED发光的状态我们用1来表示。

你很快意识到这个表示方法和二进制非常类似。现在假设有两个人，在不同的房间，一个房间只有开关，另一个房间只有LED灯，有开关房间的这一个人希望给另一个房间的人发送一个数字”5“，那么他可以做的就是，第一秒闭合开关，第二秒打开开关，第三秒闭合开关，另外一个房间的人就会接收到LED亮灭亮的信号，转换称数字标识就是”101“，再转换成十进制就是数字”5“。（当然更严谨地进行实现，发送者还应在发送之前通过某种方式提醒接收者，否则想要发送111或者100这种信号就可能产生误解，不过这不是我们这里的讨论重点）

半导体与MOS管

但是我们并不希望每次发送一个0和1都需要专门配置一个人在那摁开关，有没有一种方法可以依靠电路自身就可以控制开关呢？很幸运，人类发现了半导体，并以此研制出了MOS管，它分为NMOS和PMOS，以NMOS为例，其内部结构大概是这样：

我们可以看到N沟道场效应管（即NMOS）的漏极和源极连接着两块写着N的区域，栅极则通过一层金属和氧化物连接着NPN三块区域。具体原理有点复杂，但是你可以简单理解成N区域（N沟道）内存在大量电子，P区域（P沟道）内存在大量空穴（简单当成带正电的电子也行），正常情况下想要把两个N区域的电子进行交换需要让电子穿过P区域，但是电子会和空穴复合，一般情况下就过不去。但是如果我们给栅极一个正电压，那么N沟道里的电子就会被吸引过来，但是隔着一层氧化物薄膜，电子又过不去，最终堆积起来，在两个N沟道中间堆积形成了一座桥，这个时候漏极和源极的N沟道电子就可以流通了。

再简单抽象一下，其实就是通过控制栅极的电压，就可以控制漏极和源极的导通，就像一个电子开关一样。如果我们把可以导通开关的电压视为1，那么不能导通的电压就视为0。

逻辑门电路

现在把各种MOS电路组合一下，我们就可以搭建各种电路，而这些电路中又以逻辑电路最为重要。

所谓逻辑电路，就是可以表达逻辑运算的电路，而逻辑运算，非常巧合的是，它可以和二进制计算相联系起来。逻辑运算的主要运算类型分为与或非三种，以与运算为例，两个一位二进制数进行与运算的规则就是有0取0，没0取1（或运算是有1取1，没1取0；非运算仅针对一个一位二进制数，结果取反）。下面这个图就是表示的一个与门，A和B分别代表两个一位二进制数，Y则代表它们两个进行与运算后的输出。

假设A输入为1，B输入为0，那么Y的输出就是0。

锁存器

现在我们得到了逻辑电路的基础门电路了，试着将他们组装一下，我们不难得到一种神奇的元器件——锁存器，下面就是一个SR锁存器的示意图：

通过两个与非门接成反馈电路，S\’和R\’是输入，Q和Q\’是输出。

我们假设前一时刻，Q和Q\’的输出是1和0，那么当下一时刻S\’和R\’输入若都为1，那么Q和Q\’的结果仍然为1和0，这样就形成了一个储存电路。如果把S\’和R\’的状态与Q和Q\’的前一刻状态做一张表，最终得到的Q和Q\’输出结果可以为：

SR锁存器似乎拥有了存储的功能，至少当S\’和R\’的输入都是1时确实可以，但是任何时候我们稍微改变一下S或者R的输入，输出结果就改变了。

很显然，锁存器的变化是非同步控制的，若多个锁存器一起作为存储设备进行工作就容易产生问题。且锁存器对输入电平非常敏感，极易因电路问题产生毛刺（电平突变）而导致存储结果失效。

触发器

所以我们希望实现一种电路，能够同步电路，控制时序且过滤毛刺，而很幸运的是，可以实现且非常简单。我们只需要在一个SR锁存器前加上两个与非门，同时用一个规律方波CLK作为时钟信号控制时序，就可以得到同步SR触发器。

同步SR触发器的输出结果此时不但与S和R的输入有关，还和CLK的状态以及Q的前一刻输出有关。

例如当CLK为0时，S和R无论为任意值，都不会改变Q的输出，而当CLK为1时，假设S和R为0，则输出Q的值与前一刻Q的值一致。其余结果同样可以列出真值表：

你可能看着有点晕，但是不用在意这些0和1，毕竟触发器的种类非常多，而我们的重点是编程不是电路设计。我在这里真正想说的是，通过触发器，我们就可以实现在一定条件下，才可以根据电路上一刻的状态改变存储电路数据的值，得到新的结果。

而这个一定条件，是不是非常类似于我们最开始说的开关LED例子中每一秒整的时候才能进行开关这个条件？

事实上通过晶振（一种可以产生稳定频率震荡信号的元件）或者其它方式，我们就可以得到一个同步信号，然后在这个同步信号的控制下，触发器中的数据就能得到存储和有规律的变化。

寄存器

你应该也想到了这一点，并且敏锐地意识到我们是不是有办法把触发器组装一下，让它可以存储一些更加复杂的数据？

这是一个由四个D触发器促成的四位移位寄存器。D触发器是一种边沿触发器，当CLK的信号发生跳变的瞬间，触发器会得到触发，如果是0变到1的跳变（即上升沿，反之为下降沿），则输出Q等于输入D，如果是下降沿则保持原输出不变。

现在观察上面的四位移位寄存器，当我们假设最开始从左到右的并行输出为0000，串行输出为0。若发生第一个上升沿跳变时串行输入为1，那么跳变结束后并行输出结果为1000，串行输出为0。在之后一个周期内的保持状态和下降沿中，四个D触发器不会发生改变，输出结果也不会变。直到下一次上升沿，若之后每次上升沿时串行输入均为0，那么接下来3个周期，并行输出结果会为0100，0010，0001，串行输出为0，0，1。

不难发现，每个周期中，1都发生了向右移位，并且在移动了四次后，串行输出得到了1的输出标志。

如果要实现更加复杂的寄存器，思想也差不多是如此，只是实现方式和结果可能更加复杂多样罢了。

运算器

通过寄存器、触发器、门电路，我们就可以实现很多复杂的功能电路了。

除了我们刚才寄存器提到的移位寄存器功能，寄存器还可以组合实现存储器的功能。

而通过门电路我们也可以组成加法器，加法器可以实现二进制数的加法，通过加法器的组合，还可以组成乘法器等运算电路。

例如下图就是一个四位全加器74LS83芯片的逻辑图：

它可以完成两个4位二进制数字的加法。

如果你对冯诺依曼结构有所了解，可能会突然意识到些什么。

是的，冯诺依曼结构，也就是当今世界计算机普遍使用的计算机设计架构，指出了计算机由五个部分组成——运算器、存储器、控制器、输入设备、输出设备。

很显然，我们已经搞懂了运算器和存储器的原理，那么剩下三种又是怎么回事呢？

控制器

其实当你明白运算器和存储器的概念时，你就已经明白控制器是怎么回事了，控制器所作的工作其实就是读取指令然后根据指令输出操作指令到规定的地址。而读取指令，实际上是在做什么呢？是的，就是触发器在触发时刻接受输入的过程，而输出操作到规定的地址，其实就是控制器经过一定的运算后把指令数据处理，最终得到的输出。

不过说到这里，就不得不提一嘴地址是怎么回事了。

地址和总线

想象一下，我们在一个8位寄存器中存储了一个8位二进制数，这个二进制数可以通过8根线传输出去。我们知道，8位二进制数可以代表0~255的数，一共256（也就是2的8次方）个数。

那么，这256个数其实就可以对应到256个存储单元上，它们也就被称为这256个存储单元各自的地址，而这8根线就被称为地址总线。当计算机每次想要访问哪个存储单元的数据，只需要指定好地址，然后通过地址总线寻访到指定存储单元的数据，将其取出。

而数据取出时走的则是数据总线，地址总线长度和数据总线长度不需要相等，但是它们的设计思路是相同的。

输入设备和输出设备

关于输入设备和输出设备其实没什么好说的，输入设备就是键盘、鼠标、U盘等人从外部把数据输入到电脑中的设备，而输出设备就是屏幕、打印机这类计算机把数据表示成人能看懂的形式的设备。仍然以我们最开始的开关LED为例，输入设备就是按键开关，输出设备就是LED灯。

总结

看完这一章你可能感觉学到了很多，也可能感觉什么都没学懂，这都没关系，毕竟这只是刚开始，很多东西还没落到实处。

以上硬件基础可以方便你理解我们在后面学习的一些概念，然而事实上很多会编程的人可能对这些基础一窍不通，但是他们编程技术最终也还是不错。所以学习这些东西更多是希望你能在后面学习中不但知其然，还能知其所以然，你在学了指令和数据之后，能想明白所谓指令和数据对于电脑而言究竟是个什么东西，什么支撑着程序的运行，又是什么完成了计算等等。
